# 얼굴 인식 기능 브랜딩 및 부각 설계

## Context

CCMaxOK의 킬러 기능인 얼굴 인식(사진 → 자동 크롭 → 메뉴바 아이콘)이 현재 Settings 깊숙이 묻혀 있어 사용자가 발견하기 어렵고, 앱의 차별화 포인트로 인식되지 않는다. 이 기능을 앱 전반에 걸쳐 부각시키고 브랜딩에 활용한다.

---

## 핵심 원칙

1. **사진 한 장으로 시작하는 경험** — 복잡한 설정 없이 사진만 넣으면 완성
2. **얼굴이 앱 곳곳에 살아있어야 함** — 메뉴바 작은 아이콘에만 있으면 안 됨
3. **상태에 따라 감정이 바뀌는 느낌** — 단순 게이지가 아니라 "내 분신"이 반응

---

## Phase 1: 팝오버에 얼굴 + 상태 메시지

### 변경 사항

**팝오버 상단 프로필 영역 신설**

현재:
```
┌──────────────────┐
│     CCMaxOK      │
│ Claude Code ...  │
├──────────────────┤
│  [게이지 카드]    │
│  [통계 카드]      │
└──────────────────┘
```

변경:
```
┌──────────────────┐
│  [얼굴]  여유롭네요  │
│          85% 남음    │
├──────────────────┤
│  [게이지 카드]    │
│  [통계 카드]      │
└──────────────────┘
```

- 커스텀 이미지 미설정 시: 기존 `CCMaxOK` 헤더 유지
- 커스텀 이미지 설정 시: 얼굴 아이콘(48pt) + 상태 메시지로 교체
- 얼굴에 잔여량 기반 효과 적용 (opacity/grayscale, 기존 SimpleRenderer 로직 재사용)

**상태 메시지 (잔여량 기준)**

| 잔여 | 메시지 | 얼굴 효과 |
|---|---|---|
| 80%+ | 여유롭네요 | 선명 |
| 50~80% | 괜찮아요 | 선명 |
| 20~50% | 절반 지났어요 | 약간 흐림 |
| 10~20% | 아껴쓰세요 | grayscale |
| ~10% | 거의 다 썼어요 | grayscale + dim |

### 수정 파일

| 파일 | 작업 |
|---|---|
| `ccmaxok/Views/MenuBarView.swift` | 헤더 영역을 조건부 프로필로 교체 |
| `Sources/CCMaxOKCore/Rendering/ThemeManager.swift` | 이미지 효과 함수 재사용 |

## Phase 2: 온보딩 (첫 실행 경험)

### 변경 사항

앱 최초 실행 시 사진 설정을 유도하는 온보딩 플로우.

**팝오버 내 온보딩 카드**

첫 실행 or 커스텀 이미지 미설정 시 게이지 카드 위에 카드 표시:

```
┌──────────────────────┐
│  📸 사진 한 장으로     │
│  나만의 메뉴바를       │
│  만들어보세요          │
│                       │
│  [사진 선택하기]       │
│         건너뛰기       │
└──────────────────────┘
```

- "사진 선택하기" → NSOpenPanel → 얼굴 자동 감지 + 크롭 → 바로 적용
- "건너뛰기" → `UserDefaults("onboarding_dismissed")` 저장, 다시 안 뜸
- Settings에서 언제든 설정 가능하므로 강제하지 않음

### 수정 파일

| 파일 | 작업 |
|---|---|
| `ccmaxok/Views/MenuBarView.swift` | 온보딩 카드 조건부 표시 |
| `ccmaxok/Views/SettingsView.swift` | 이미지 선택 로직 재사용 (함수 분리 필요할 수 있음) |

## Phase 3: 얼굴 기반 상태 연출 강화

### 변경 사항

단일 이미지에서 더 풍부한 상태 표현.

**방법 A: 이미지 효과 강화 (코드 기반, 추가 이미지 불필요)**

| 잔여 | 효과 |
|---|---|
| 80%+ | 원본 + 미세한 glow (밝기 +10%) |
| 50~80% | 원본 그대로 |
| 20~50% | saturation 50% |
| ~20% | grayscale + opacity 60% |

**방법 B: 이모지 오버레이 (이미지 위에 상태 이모지)**

| 잔여 | 오버레이 |
|---|---|
| 80%+ | 😊 (우측 하단 작게) |
| 50~80% | 없음 |
| 20~50% | 😐 |
| ~20% | 😵 |

**추천**: 방법 A가 깔끔. 이모지 오버레이는 취향 갈림.

### 수정 파일

| 파일 | 작업 |
|---|---|
| `Sources/CCMaxOKCore/Rendering/ThemeManager.swift` | saturation/brightness 효과 추가 |
| `Sources/CCMaxOKCore/Rendering/SimpleRenderer.swift` | 효과 적용 분기 세분화 |

## Phase 4: 브랜딩 연동

### 앱 이름 변경 시 연동 포인트

만약 "FaceGauge", "FaceMeter" 등으로 이름을 바꾼다면:

1. **앱 아이콘**: 얼굴 실루엣 + 게이지 원형 조합
2. **팝오버 헤더**: 앱 이름 대신 사용자 얼굴이 브랜드 역할
3. **미설정 시 기본 아이콘**: 실루엣 얼굴(👤 스타일)을 기본 아이콘으로 사용 → "여기에 내 얼굴을 넣으세요" 느낌
4. **Homebrew 설명**: `"Your face as a Claude Code usage meter"` 

### 기본 아이콘 전략

커스텀 이미지 미설정 시 `⬤` 대신 `👤` 실루엣을 기본으로 하면:
- 사용자가 "여기에 내 사진을 넣을 수 있구나" 직감
- 사진 넣은 후 "내 얼굴로 바뀌었다!" 감동 포인트

## 구현 우선순위

| 순서 | 내용 | 효과 |
|---|---|---|
| **1** | 팝오버 얼굴 + 상태 메시지 (Phase 1) | 즉시 차별화 체감 |
| **2** | 온보딩 카드 (Phase 2) | 기능 발견율 ↑ |
| **3** | 상태 연출 강화 (Phase 3) | 감성적 연결 |
| **4** | 브랜딩 연동 (Phase 4) | 앱 이름 확정 후 |

## 기술 참고

- 얼굴 이미지 로드: `ThemeManager.shared.loadImage(named: imageHigh)` (기존)
- 마스크 적용: `FaceCropper.applyShapeMask(to:mask:)` (기존)
- 효과 적용: `ThemeManager.shared.applyOpacity/applyGrayscale` (기존)
- 잔여량: `AppState.fiveHourUsedPct` → `100 - usedPct`
- 이미지 설정 여부: `UserDefaults.standard.string(forKey: "image_high")`
